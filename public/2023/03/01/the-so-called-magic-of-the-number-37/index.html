But let me tell you, even $15873$ is super-weak compared to other numbers that carry uber-highly-amplified and/or uber-highly-vibrational powers. So let’s write a small Python script that can list more vibrationally powerful numbers (whatever that is) than $37$ and $15873$.

from primefac import primefac
from collections import Counter
import sys

def compute_vibrational_num(vp):
    n, res, vp_ret = 1, 0, vp
    while vp != 0: n, vp = 1000 * n + 1, vp - 1
    n *= 111
    primef = Counter(list(primefac(n))).most_common(2)
    primef = primef[1 if primef[1][0] == 7 else 0]
    magic = primef[0] ** primef[1]
    res=n//magic
    return (magic, int(res), vp_ret)

Repdigits are natural numbers composed of instances of the same digit. The coolest repdigits are the Mersenne primes.

They are prime numbers that, when represented in binary, are composed entirely of 1s.

All in all, let’s get back to our powerful vibrational numbers (whatever that means).

Numbers like $11…1$ can sometimes be prime, but if the number of 1 digits is a multiple of 3, we know for a fact that they aren’t. Here is a simple proof:

$$ \underbrace{111...111}_{\text{3i digits}} = 10^0 + 10^1 + 10^2 + ... + 10^{3i-3} + 10^{3i-2} + 10^{3i-1} $$

If we group the digits in sets of three, we get the following relationship:

$$ 111...111 = \underbrace{(1 + 10 + 100)}_{111} + ... + 10^{3i-3}*\underbrace{(1 + 10 + 100)}_{111} $$

And then if we use $111$ as a common factor, we obtain:

$$ 111...111 = 111 * (1 + 10^3 + ... + 10^{3i-3}) $$

But $111 = 3 \times 37$, so repdigits with a number of digits that is a multiple of 3 are always divisible by 3 and 37. As an interesting observation, they are from time to time divisible by 7 (but not always).

This is exactly how our code functions:

  • It generates numbers like $111..111$;
  • It gets their prime factors (we know for certain they are not prime);
  • Then it separates the prime factors into a “bigger”, highly vibrational number, and keeps a multiple of 3 or 7 as the meme multiplier.
